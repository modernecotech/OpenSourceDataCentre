# Project Lifecycle Gates

Project gates prevent the design from moving forward on enthusiasm alone.

Gate sequence:

1. Initiation boundary.
2. Site selection.
3. Concept design.
4. Design freeze.
5. Procurement release.
6. Construction readiness.
7. Commissioning acceptance.
8. Operational readiness.
9. Customer readiness.
10. Handover to operations.

Each gate must record:

- owner;
- exit criteria;
- required evidence;
- unresolved risks;
- accepted exceptions;
- approval date;
- next actions.
