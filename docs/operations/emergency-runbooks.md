# Emergency Runbooks

Emergency runbooks must be usable when the portal, internet, or vendor support path is unavailable.

## Minimum Runbooks

- Grid loss.
- Generator start failure.
- DC bus undervoltage.
- Cooling loop failure.
- Leak detection.
- Fire alarm.
- Access-control outage.
- Identity outage.
- Ceph degraded storage.
- Kubernetes control-plane failure.
- Backup restore.
- Security incident.

## Runbook Format

- Symptom.
- Immediate safety action.
- Who to call.
- Systems affected.
- Checks to perform.
- Decision points.
- Recovery procedure.
- Verification test.
- Evidence to capture.
- Follow-up actions.
