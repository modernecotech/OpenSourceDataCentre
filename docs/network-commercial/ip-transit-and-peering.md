# IP Transit and Peering

IP transit and peering turn the facility into part of national network infrastructure.

Define:

- ASN ownership;
- upstream providers;
- route filters;
- RPKI/ROA policy;
- prefix limits;
- DDoS escalation path;
- blackhole or FlowSpec policy;
- route-server policy for IXP services;
- maintenance windows;
- customer demarcation.

Route changes should use MOPs and rollback plans.
