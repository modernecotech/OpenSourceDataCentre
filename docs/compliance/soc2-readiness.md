# SOC 2 Readiness

This file tracks readiness for SOC 2-style evidence. It is not an audit report.

Trust-service areas to map:

- security;
- availability;
- processing integrity where platform workflows are in scope;
- confidentiality where sensitive tenant data is handled;
- privacy where personal data is in scope.

Evidence examples:

- access reviews;
- change approvals;
- incident records;
- vulnerability scans;
- backup restore tests;
- capacity and availability reports;
- customer communication records;
- supplier risk reviews;
- physical access logs;
- CCTV retention policy.

Readiness rule:

Every control must have an owner, frequency, evidence path, exception process, and management review path.
