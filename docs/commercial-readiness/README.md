# Commercial Readiness

This section tracks the difference between an OSDC reference architecture and a commercial-grade colocation or sovereign cloud datacentre.

The repository must stay honest: these files are readiness templates and gap trackers, not certification claims, legal contracts, or signed engineering deliverables.

Core artifacts:

- [Gap register](gap-register.md)
- [Standards compliance matrix](standards-compliance-matrix.md)
- [Uptime/Tier gap analysis](uptime-tier-gap-analysis.md)
- [TIA-942 gap analysis](tia-942-gap-analysis.md)
- [EN 50600 / ISO 22237 gap analysis](en50600-iso22237-gap-analysis.md)
- [Certification boundary](certification-boundary.md)
- [Tier gap roll-up](tier-gap-analysis.md)
- [SLA model](sla-model.md)
- [Commercial service catalogue](commercial-service-catalogue.md)
- [Audit evidence register](audit-evidence-register.md)
- [Customer responsibility matrix](customer-responsibility-matrix.md)

Machine-readable data lives in [data/commercial](../../data/commercial/).

Commercial readiness depends on five evidence families:

- standards and certification mapping;
- MEP engineering calculations and drawings;
- operating procedures and staffing model;
- customer products, SLAs, and responsibility boundaries;
- audit evidence and continual improvement.
