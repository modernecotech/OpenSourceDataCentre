# Licensing

This repository is intended to be open source and open design. Unless a file
states otherwise, use the following licence split:

| Material | Licence | SPDX identifier |
| --- | --- | --- |
| Rust crates, scripts, tests, and prototype application code | Apache License 2.0 | `Apache-2.0` |
| Documentation, CSV/JSON planning data, examples, diagrams, and training material | Creative Commons Attribution 4.0 International | `CC-BY-4.0` |
| Hardware and mechanical design source files, including future CAD, IFC, STEP, FreeCAD, rack, enclosure, thermal, and power-distribution design artifacts | CERN Open Hardware Licence Version 2 - Strongly Reciprocal | `CERN-OHL-S-2.0` |

The Rust workspace publishes `Apache-2.0` through `Cargo.toml`.

Canonical licence texts:

- Apache-2.0: https://www.apache.org/licenses/LICENSE-2.0
- CC-BY-4.0: https://creativecommons.org/licenses/by/4.0/legalcode
- CERN-OHL-S-2.0: https://ohwr.org/cern_ohl_s_v2.txt

Documentation and data can be copied, adapted, translated, and redistributed
with attribution under CC-BY-4.0. Hardware design files should preserve source
availability and reciprocal open-hardware obligations under CERN-OHL-S-2.0.

This file is a repository policy statement, not legal advice. Downstream
deployments should confirm licence compatibility with their own legal and
procurement teams before using the project in regulated infrastructure.
